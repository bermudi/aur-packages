# AGENTS.md

## Project
Monorepo for bermudi's AUR packages. One GitHub repo = source of truth;
GitHub Actions keep each AUR package in sync. Packages build from upstream
APT deb repositories provided by Codeium (Devin Editor).

## Architecture
```
<pkgname>/              # one dir per AUR package
  PKGBUILD              # source of truth (in-place edited by CI, never regenerated)
  .SRCINFO              # generated by .github/scripts/generate-srcinfo.sh
  *.desktop             # patched for /opt relocation
.github/
  scripts/
    update-pkgbuild.sh  # in-place: only pkgver, sha256sums[0], _apt_pool
    generate-srcinfo.sh # sources PKGBUILD -> .SRCINFO (no makepkg needed on CI)
  workflows/
    <pkgname>.yml       # per-pkg: poll upstream, compare, call update-aur
    update-aur.yml      # reusable: bump, commit, release, push to AUR (clone-or-init)
```

**Update pipeline:** poll APT Packages file -> parse version + sha256 + filename ->
convert to Arch version format -> compare against repo PKGBUILD ->
if different (or AUR pkg missing) -> call reusable updater -> commit + release -> push to AUR.

AUR push uses **clone-or-init**: if the AUR repo doesn't exist yet, it's created on first push.
This is why the existence gate (query AUR RPC) is important — it forces the initial seed
when `resultcount:0`.

## Stable Reference Facts

### Upstream APT channels
Both hosted at `windsurf-stable.codeiumdata.com` (yes, the hostname still says "windsurf" post-rebrand).

| channel | APT token (URL path segment) | dist | Arch pkg |
|---------|------------------------------|------|----------|
| next    | `mQfcApCOdSLoWOSI`           | `dists/next/main/binary-amd64/Packages`    | `devin-desktop-next` |
| stable  | `wVxQEIWkwPUEAGf3`           | `dists/stable/main/binary-amd64/Packages`  | (not ours — `devin-desktop` maintained by luizsvdev) |

### Version conversion
APT: `3.1.1005+next.296eca6010-1780977384` → Arch: `3.1.1005_next.296eca6010`
Rule: `+` → `_`, then strip everything after the first `-` (Debian revision).

### AUR
- SSH push: `ssh://aur@aur.archlinux.org/<pkgname>.git` (default branch: `master`)
- The `AUR_SSH_PRIVATE_KEY` repo secret must be set for AUR push to work
- New packages: first `git push` creates the package; `git clone` of nonexistent pkg fails — use clone-or-init

### PKGBUILD conventions
- Packages install to `/opt/<pkgname>/` (not `/usr/share/` as the upstream deb does)
- Desktop files are patched to point at `/opt/` paths (the deb ships its own but they'd be wrong)
- Binary symlink: `/usr/bin/<pkgname>` → `/opt/<pkgname>/<pkgname>`
- Icon sourced from `<installdir>/resources/app/resources/linux/code-next.png`
- Upstream packages inside the deb: `usr/share/devin-desktop-next/` (next) or `usr/share/devin-desktop/` (stable)

## Conventions

**Never regenerate the entire PKGBUILD from a template in CI.** The `update-pkgbuild.sh`
script edits only `pkgver`, the first `sha256sums` entry (the deb), and `_apt_pool`
if upstream relocated the file. Everything else — depends, `package()`, desktop file
references — is hand-maintained and must survive automated bumps intact.

**.SRCINFO is always generated**, never hand-edited. The generator pre-declares all
PKGBUILD arrays as empty to handle optional fields cleanly. Validate the output is
byte-identical to `makepkg --printsrcinfo` after changes.

## Adding a Package

1. Create `<pkgname>/` with `PKGBUILD`, `.SRCINFO`, and auxiliary files.
2. Add `.github/workflows/<pkgname>.yml` modelled on the existing checker.
3. First run seeds the AUR package automatically (existence gate -> `has_update=true`).

## Constraints

- **Don't create a `devin` or `devin-desktop` package.** `devin-desktop` already exists
  on AUR (maintained by luizsvdev). This repo only manages the next channel.
- **Don't exfiltrate SSH private keys.** Use `< file` redirection for `gh secret set`.
- **Validate before pushing AUR-side changes.** At minimum: `bash -n PKGBUILD` and
  confirm `.SRCINFO` matches `makepkg --printsrcinfo`. The CI does this automatically;
  make sure it passes before merging.
- **No build artifacts in the repo.** `src/`, `pkg/`, `*.deb`, `*.pkg.tar.*` are
  gitignored. Don't commit them; don't un-ignore them.
